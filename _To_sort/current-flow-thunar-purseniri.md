The current flow (Thunar → purse-niri → Purse) handles ingestion well, but dispatch is still a blunt instrument — Enter opens files in their "next best" handler, which is functional but not intentional. The natural next evolution is a verb picker / launcher: once files are staged in Purse, you'd invoke something that says "now what?" — open in X, send to Y, run Z on them. This could live as a separate binary that Purse hands off to via socket, keeping the same pattern as purse-niri.

 A second thread is ingestion UX. Thunar works but it's incidental — you're using a general-purpose file manager with purse-niri bolted on as default handler. A purpose-built lightweight file picker that knows it's feeding a Purse would be tighter: navigate, throw things in, done. No right-click ambiguity, no MIME-handler gymnastics.
